## Introduzione: Le Schede Didattiche come Strumenti di Valutazione

**Tipologie di valutazione**:
- **Formativa**: Durante l'apprendimento (monitora i progressi, adatta l'insegnamento)
- **Sommativa**: Dopo l'apprendimento (valuta la padronanza, assegna i voti)
- **Diagnostica**: Prima dell'apprendimento (identifica le lacune, pianifica l'intervento)

**Il vantaggio delle schede didattiche**: Documentazione integrata (registro fisico delle prestazioni dell'alunno)

**Il problema della valutazione tradizionale**:
```
Valutazione orale dell'insegnante: "Quanto fa 5 + 3?"
Alunno: "8"
Insegnante: ✓ (nota mentale: l'alunno conosce l'addizione)
Problema: Nessuna traccia permanente, impossibile monitorare i progressi nel tempo
```

**La soluzione con le schede didattiche**:
```
Settimana 1: Scheda addizioni (10 problemi), punteggio: 60% (6/10 corretti)
Settimana 4: Scheda addizioni (10 problemi), punteggio: 80% (8/10 corretti)
Settimana 8: Scheda addizioni (10 problemi), punteggio: 100% (10/10 corretti)
Documentazione: Chiara traiettoria di progresso (60% → 80% → 100%)
```

**Ricerca** (Black & Wiliam, 1998): La valutazione formativa migliora i risultati di apprendimento di **0,4-0,7 deviazioni standard** (equivalente a spostare uno studente dal 50° al 65-75° percentile)

**Requisito fondamentale**: Raccolta dati sistematica (le schede didattiche forniscono questo)

## Valutazione Formativa con i Generatori

### Strategia 1: Monitoraggio Settimanale dei Progressi

**Concetto**: Stessa competenza, misurazione settimanale

**Implementazione (esempio matematica)**:
```
Competenza: Calcoli di addizione 1-20

Settimana 1: Genera scheda addizioni (20 problemi, intervallo 1-20)
            L'alunno completa: 12/20 corretti (60%)

Settimana 2: Genera NUOVA scheda addizioni (problemi diversi, stessa competenza)
            L'alunno completa: 15/20 corretti (75%)

Settimana 3: Genera NUOVA scheda
            L'alunno completa: 18/20 corretti (90%)

Settimana 4: Genera NUOVA scheda
            L'alunno completa: 20/20 corretti (100%) ✓ PADRONANZA ACQUISITA

Dati: Chiara traiettoria ascendente (l'intervento sta funzionando)
```

**Vantaggio del generatore**: Problemi sempre nuovi ogni settimana (nessuna memorizzazione, vera valutazione della competenza)

**Migliori generatori per il monitoraggio progressi**:
- Schede di Matematica (App 010): Valutazione settimanale delle operazioni matematiche
- Addizioni/Sottrazioni (App 002/007): Tracciamento di operazioni specifiche
- Crucipuzzle (App 003): Velocità di riconoscimento lessicale (tempo di completamento)

---

### Strategia 2: Analisi degli Schemi di Errore

**Concetto**: Identificare i fraintendimenti specifici (non solo "sbagliato")

**Implementazione (esempio matematica)**:
```
L'alunno completa una scheda di sottrazioni:
- 15 - 7 = 12 (errato, dovrebbe essere 8)
- 14 - 6 = 12 (errato, dovrebbe essere 8)
- 13 - 5 = 12 (errato, dovrebbe essere 8)

Schema identificato: L'alunno ottiene sempre 12 (sta sommando invece di sottrarre)
Fraintendimento: Confusione tra operazioni (+/−)
Intervento mirato: Rivedere il concetto di sottrazione con materiali concreti
```

**Categorie di analisi degli errori**:

**Errori di calcolo**: Risposta sbagliata, ma procedura corretta
- 7 + 8 = 14 (dovrebbe essere 15, errore di conteggio)

**Errori concettuali**: Fraintendimento fondamentale
- 7 + 8 = 78 (concatena le cifre, non sta sommando)

**Errori procedurali**: Procedura sbagliata
- Sottrae il minore dal maggiore indipendentemente dalla posizione (42 - 28 = 26, dovrebbe essere 14)

**Beneficio del generatore**: Il formato coerente permette di individuare facilmente gli schemi

### Strategia 3: Valutazione Differenziata

**Concetto**: Valutare al livello attuale dell'alunno (non una taglia unica per tutti)

**Problema delle valutazioni standard**:
```
Verifica per tutta la classe: 20 problemi di addizione (intervallo 1-100)
- Alunno avanzato: Finisce in 3 minuti (troppo facile, nessuna sfida)
- Alunno livello medio: Finisce in 10 minuti (appropriato)
- Alunno in difficoltà: Non riesce a completare (troppo difficile, frustrazione)
Risultato: La valutazione non è utile per 2/3 degli alunni
```

**Soluzione con il generatore**:
```
Stessa competenza (addizione), difficoltà differenziata:

Alunno in difficoltà: Addizioni 1-20, modalità immagini, 10 problemi
                      Completa con successo (sfida appropriata)

Alunno livello medio: Addizioni 1-100, modalità simbolica, 20 problemi
                     Difficoltà appropriata

Alunno avanzato: Addizioni 100-1000, multi-cifra, 25 problemi
                Sfida appropriata

Risultato: Tutti gli alunni vengono valutati al loro livello (dati significativi per tutti)
```

**Tempo per creare valutazioni differenziate**:
- Manuale: 3 valutazioni × 30 min = 90 minuti
- Generatori: 3 valutazioni × 42 sec = 126 secondi (2,1 minuti)
- **Tempo risparmiato**: 87,9 minuti per evento di valutazione

## Monitoraggio dei Progressi per gli Obiettivi PEI

**Requisito PEI**: Documentare i progressi verso gli obiettivi (trimestrale o più frequente)

**Vantaggio del generatore**: Strumento di misurazione coerente

**Esempio di obiettivo PEI**: "L'alunno risolverà 15/20 problemi di addizione a una cifra con l'80% di precisione entro maggio 2025"

**Raccolta dati usando i generatori**:

**Baseline di settembre**:
```
Genera: Scheda addizioni (20 problemi, intervallo 1-10)
Prestazione alunno: 8/20 corretti (40%)
Nota PEI: Al di sotto dell'obiettivo (40% vs 80% target)
```

**Verifica progressi novembre**:
```
Genera: NUOVA scheda addizioni (20 problemi, stesso intervallo)
Prestazione alunno: 12/20 corretti (60%)
Nota PEI: In miglioramento (40% → 60%, +20 punti percentuali)
```

**Verifica progressi gennaio**:
```
Genera: NUOVA scheda
Prestazione alunno: 15/20 corretti (75%)
Nota PEI: Vicino all'obiettivo (75% vs 80% target)
```

**Verifica progressi marzo**:
```
Genera: NUOVA scheda
Prestazione alunno: 17/20 corretti (85%)
Nota PEI: OBIETTIVO RAGGIUNTO (85% supera il target dell'80%) ✓
```

**Incontro PEI annuale**: Presentare tutte le 4 schede (documentazione visiva dei progressi)

**Conformità normativa**: Registro permanente (richiesto per la documentazione PEI)

## Dati per la Risposta all'Intervento (RTI)

**Livelli RTI**:
- **Livello 1**: Insegnamento per l'intera classe (80% degli alunni ha successo)
- **Livello 2**: Intervento in piccoli gruppi (15% necessita supporto extra)
- **Livello 3**: Intervento individuale intensivo (5% necessita aiuto specializzato)

**Requisito RTI**: Monitoraggio frequente dei progressi (settimanale o bisettimanale)

**Applicazione del generatore**:

**Alunno Livello 2** (riceve intervento matematico in piccoli gruppi):
```
Settimana 1: Scheda addizioni, 50% precisione (inizia l'intervento)
Settimana 2: Scheda addizioni, 55% precisione (leggero miglioramento)
Settimana 3: Scheda addizioni, 60% precisione (crescita costante)
Settimana 4: Scheda addizioni, 65% precisione (risponde all'intervento)
Settimana 5: Scheda addizioni, 70% precisione
Settimana 6: Scheda addizioni, 75% precisione
Settimana 7: Scheda addizioni, 80% precisione
Settimana 8: Scheda addizioni, 85% precisione (esce dal Livello 2, ritorna al Livello 1)

Dati: 8 punti dati in 8 settimane (sufficienti per le decisioni RTI)
Risultato RTI: Intervento riuscito, l'alunno esce dal Livello 2
```

**Frequenza**: Settimanale (i generatori rendono questo fattibile, la creazione manuale richiederebbe 8 × 40 min = 320 minuti)

## Valutazioni di Riferimento

**Valutazione di riferimento**: 3 volte all'anno (inizio, metà, fine anno)

**Scopo**: Misurare la crescita anno dopo anno

**Implementazione**:

**Settembre (Inizio anno)**:
```
Valutazione di riferimento matematica 2ª elementare:
- Addizioni 1-20: Genera 20 problemi
- Sottrazioni 1-20: Genera 20 problemi
- Punteggi alunno: Addizioni 60%, Sottrazioni 50%
Baseline stabilita
```

**Gennaio (Metà anno)**:
```
Stesse competenze, NUOVI problemi (i generatori creano nuove valutazioni):
- Addizioni 1-20: NUOVI 20 problemi
- Sottrazioni 1-20: NUOVI 20 problemi
- Punteggi alunno: Addizioni 85%, Sottrazioni 75%
Progresso: +25% addizioni, +25% sottrazioni (sulla buona strada)
```

**Maggio (Fine anno)**:
```
Stesse competenze, NUOVI problemi:
- Addizioni 1-20: NUOVI 20 problemi
- Sottrazioni 1-20: NUOVI 20 problemi
- Punteggi alunno: Addizioni 95%, Sottrazioni 90%
Crescita annuale: +35% addizioni, +40% sottrazioni (forte crescita)
```

**Visualizzazione dati**:
```
Progresso addizioni: 60% → 85% → 95% (chiara traiettoria di crescita)
Progresso sottrazioni: 50% → 75% → 90% (forte miglioramento)
Conclusione: L'alunno ha fatto eccellenti progressi (da riferire ai genitori/dirigenza)
```

## Decisioni Didattiche Basate sui Dati

**Utilizzare i dati di valutazione per adattare l'insegnamento**:

**Scenario 1**: L'80% della classe ha difficoltà con il prestito nelle sottrazioni
```
Dati: Media classe sulla scheda sottrazioni: 45%
Schema errori: La maggior parte degli errori sui problemi che richiedono il prestito (23 - 17, 42 - 28)
Decisione: Rivedere il concetto di prestito (non solo assegnare più esercizi)
Azione: Usare materiali concreti, poi generare nuova scheda per rivalutare
```

**Scenario 2**: Alunno avanzato mostra noia
```
Dati: L'alunno completa la scheda matematica di 2ª in 3 minuti, 100% precisione
Decisione: L'alunno necessita materiale avanzato
Azione: Genera scheda di livello 4ª elementare (sfida appropriata)
Risultato: Alunno coinvolto, lavora a livello appropriato
```

**Scenario 3**: Regressione dell'alunno
```
Dati: Punteggi dell'alunno in calo (Settimana 1: 80%, Settimana 2: 75%, Settimana 3: 65%)
Segnale d'allarme: La regressione indica un problema (non sta apprendendo, o fattore esterno)
Decisione: Indagare (parlare con l'alunno, i genitori, il consulente)
Azione: Potrebbe necessitare intervento, adattamenti o supporto
```

**I dati informano le decisioni**: I generatori forniscono dati rapidamente e coerentemente
