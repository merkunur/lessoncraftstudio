## Wie funktioniert Code-Addition?

### Das Verschlüsselungssystem

**Schritt 1: Codeschlüssel erstellen**
- Jeder Zahl (0-20) wird ein Buchstabe zugeordnet
- Beispiel: 1=A, 2=B, 3=C, 4=D, 5=E, 6=F, 7=G, 8=H...

**Schritt 2: Rechenaufgaben generieren**
- Aufgaben erstellen, deren Summen zum Codeschlüssel passen
- Beispiel: Um "KATZE" zu buchstabieren, brauchen wir Summen von 11, 1, 20, 26, 5
- Generiert: 5+6=11 (K), 0+1=1 (A), 15+5=20 (T), 13+13=26 (Z), 2+3=5 (E)

**Schritt 3: Schüler löst**
- Berechnet jede Summe
- Nutzt Codeschlüssel zur Entschlüsselung
- Schreibt Buchstaben in die Lücke
- Setzt Buchstaben zum Lösungswort zusammen

**Schritt 4: Selbstkontrolle**
- Ergibt das Lösungswort Sinn? (KATZE ist ein echtes Wort)
- Stimmt die Rechnung? (Kontrolle mit Lösungsblatt)

### Vorteil 2: Automatisierung des Einmaleins

**Klassisches Rechentraining**:
- Arbeitsblatt: 50 Additionsaufgaben
- Schüler bearbeitet in 8 Minuten
- Behaltensleistung: Gering (kein sinnvoller Kontext)

**Code-basierte Übung**:

**Warum Kontext hilft** (Transfer-angemessene Verarbeitung):
- Gehirn kodiert Rechenaufgabe + Rätselkontext
- Multiple Abrufhinweise ("Das war die KATZE-Aufgabe")
- Tiefere Verarbeitung = stärkeres Gedächtnis

**Forschung** (Bransford et al., 1989): Bedeutsamer Kontext verbessert Behaltensleistung um das 2- bis 3-fache gegenüber isoliertem Üben

### Vorteil 3: Fehlerkennung (Selbstkontrolle)

**Klassisches Arbeitsblatt**:
- Schüler rechnet 3 + 4 = 6 (Fehler)
- Macht weiter mit nächster Aufgabe
- Fehler bleibt unentdeckt bis zur Lehrerkontrolle

**Code-Arbeitsblatt**:
- Schüler rechnet 3 + 4 = 6
- Entschlüsselt Buchstaben mit Codeschlüssel: 6 = F
- Macht weiter: Nächste Buchstaben ergeben "FAR" statt "KATZE"
- Schüler denkt: "FAR ist kein Wort... Ich habe einen Fehler gemacht!"
- **Korrigiert selbst vor Abgabe**

**Ergebnis**: Eingebaute Fehlererkennung (wenn entschlüsseltes Wort keinen Sinn ergibt, überprüfe deine Rechnung)

**Forschung**: Unmittelbare Feedback-Schleifen reduzieren Fehlerpersistenz um 76% (Hattie & Timperley, 2007)

### Vorteil 4: Fächerübergreifendes Lernen

**Mathe + Rechtschreibung**:
- Übe Additionen
- Festige Rechtschreibung von Lernwörtern
- "Das Lösungswort ist FREUNDSCHAFT – kannst du es richtig schreiben?"

**Mathe + Sachunterricht**:

**Mathe + Deutsch**:

### Schritt 3: Generieren (2 Sekunden)

**Algorithmus**:
1. Wandelt Wörter in Zahlenwerte mittels Code um
2. Generiert Additionsaufgaben mit korrekten Summen

4. Erstellt Codeschlüssel-Tabelle
5. Generiert Lösungsblatt

---

### Schritt 4: Optionale Bearbeitung (15 Sekunden)

**Nachbearbeitungsoptionen**:
- Einzelne Aufgaben austauschen (andere Summanden, gleiche Summe)
- Schriftgröße anpassen
- Wörter neu ordnen
- Eigene Arbeitsanweisungen hinzufügen

### Schritt 5: Exportieren (3 Sekunden)

**Formate**: PDF oder JPEG

## Einsatz im Unterricht

### Strategie 1: Geheimbotschaft-Montag

**Wochenritual**:
- Lehrkraft schreibt Code-Addition an die Tafel
- Schüler entschlüsseln beim Ankommen (5-8 Minuten)
- Botschaft verrät Freitagsaktivität
- Beispiel: "Am Freitag gehen wir ins Museum!"

**Engagement**: 96% Teilnahme (intrinsische Neugier treibt Bearbeitung an)

---

### Strategie 2: Agenten-Akademie

**Mehrwöchige Einheit**:
- **Woche 1**: "Agentenausbildung Stufe 1" (Sehr leichte Codes)
- **Woche 2**: "Agentenausbildung Stufe 2" (Leichte Codes)
- **Woche 3**: "Agentenausbildung Stufe 3" (Mittlere Codes)
- **Woche 4**: "Abschlussmission" (Schwieriger Code mit Satzbotschaft)

**Motivation**: Schüler verdienen "Agenten-Abzeichen" für Fertigstellung

**Lerneffekt**: 120 Additionsaufgaben in spielerischem Kontext geübt

---

### Strategie 3: Partner-Entschlüsselung

**Aufbau**:
- Schüler A löst Rechenaufgaben
- Schüler B nutzt Codeschlüssel zur Buchstabenentschlüsselung
- Partner arbeiten zusammen an Lösungswort
- Rollentausch beim nächsten Arbeitsblatt

**Vorteile**:
- Peer-Teaching (gegenseitige Fehlerkontrolle)
- Arbeitsteilung (berücksichtigt unterschiedliche Leistungsniveaus)
- Soziales Lernen

---

### Strategie 4: Eigenen Code erstellen

**Fortgeschrittene Erweiterung** (ab 3. Klasse):

**Aufgabe**:
1. Schüler erstellt persönlichen Codeschlüssel (A=5, B=12, C=3, usw.)
2. Schreibt Geheimbotschaft
3. Entwickelt rückwärts Additionsaufgaben (Summe muss zum Code passen)
4. Tauscht mit Partner
5. Partner entschlüsselt

**Höhere Denkleistung**: Code erstellen erfordert Rückwärtsdenken (Syntheseebene nach Bloom)

## Differenzierungsstrategien

### Für leistungsschwächere Schüler

**Anpassungen**:
- Nur sehr leichte Stufe (Summen 0-10)
- Maximal 3-Buchstaben-Wörter
- Codeschlüssel sichtbar auf derselben Seite
- Ersten Buchstaben als Beispiel vorausfüllen
- Taschenrechner erlaubt (Fokus auf Code-Logik, nicht Rechnung)

**Ziel**: Vertrauen mit Rätselformat aufbauen, bevor mathematische Schwierigkeit steigt

---

### Für leistungsstarke Schüler

**Erweiterungen**:
- Zweistellige Addition (ohne Codeschlüssel, muss aus Kontext erschlossen werden)
- Mehrsatz-Botschaften (50+ Aufgaben)
- Code mit Subtraktion statt Addition erstellen
- Code erstellen, bei dem mehrere Rechenarten (+ und −) vorkommen

**Herausforderung**: "Kannst du einen unlösbaren Code erstellen?" (lernen über Code-Validierung)

### Müssen Schüler das kleine Einmaleins können für Code-Addition?

### Kann ich Codes in anderen Sprachen erstellen?

**Beispiel**: Französischlehrkraft erstellt Code mit "Le chat est noir" (Die Katze ist schwarz)

### Was, wenn Schüler einfach das Wort raten ohne zu rechnen?

**Präventionsstrategien**:
1. Ungewöhnliche Wörter verwenden (nicht immer KATZE, HUND, BAUM)
2. Buchstabenreihenfolge mischen (nicht alphabetisch gehen)
3. Rechenweg verlangen (Summe für jede Aufgabe notieren)
4. Satzlänge-Botschaften verwenden (unmöglich zu erraten)

**Bewertung**: Rechenweg auf Lösungsblatt prüfen (haben sie gerechnet oder geraten?)
