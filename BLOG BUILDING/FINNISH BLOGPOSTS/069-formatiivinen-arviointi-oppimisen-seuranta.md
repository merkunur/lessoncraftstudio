## Johdanto: Arviointi oppimisen tukena vai oppimisen mittaamisena

**Kaksi arviointityyppiä**:
- **Summatiivinen arviointi**: Oppimisen mittaaminen (loppukoe, mitä opittu)
- **Formatiivinen arviointi**: Oppimisen tukeminen (jatkuva seuranta, ohjaa opetusta)

**Tutkimus** (Black & Wiliam, 1998): Formatiivinen arviointi parantaa oppimistuloksia **0,4-0,7 keskihajontaa** (vastaa siirtymistä 50. persentiilistä 66.-77. persentiilin tasolle)

**Perinteisen arvioinnin ongelma**:
```
Viikot 1-3: Opetetaan yhteenlaskua
Viikko 4: Koe (summatiivinen arviointi)
Tulos: Havaitaan kokeessa, että 30 % oppilaista ei ymmärtänyt
Ongelma: Liian myöhäistä - siirrytty jo seuraavaan aiheeseen
Seuraus: Oppilaat jäävät jälkeen (puutteet kasaantuvat)
```

**Formatiivisen arvioinnin ratkaisu**:
```
Viikko 1, päivä 1: Opetetaan yhteenlaskun käsite
Viikko 1, päivä 2: Nopea tehtävämonistetarkistus (5 tehtävää)
Viikko 1, päivä 2 (iltapäivä): Analysoidaan tulokset
Viikko 1, päivä 3: Uudelleenopetus vaikeuksia kokeneille (tuki)
Viikko 1, päivä 4: Tarkistetaan uudelleen (oppimisen seuranta)
Tulos: Väärinkäsitykset havaitaan OPPIMISEN AIKANA (ennaltaehkäistään puutteet)
```

**Keskeinen ero**: Formatiivinen = OPETUKSEN AIKANA (voidaan mukauttaa), Summatiivinen = OPETUKSEN JÄLKEEN (liian myöhäistä)

## Päivittäinen oppimisen seurantajärjestelmä

**Periaate**: Lyhyet päivittäiset tarkistukset (5-10 minuuttia) seuraavat oppimista reaaliajassa

### 5-tehtävän tarkistus

**Rakenne**:
```
Päivittäinen mini-moniste (luotu 42 sekunnissa):
- 5 tehtävää, jotka testaavat nykyistä taitoa
- Oppilaat tekevät 5 minuutissa
- Opettaja tarkastaa 5 minuutissa
- Kokonaisaika: 10 minuuttia (sisältää opetuksen mukauttamisen)
```

**Esimerkki: Viikko, jolloin opetetaan 2-numeroisia yhteenlaskuja**

**Maanantain mini-tarkistus**:
```
5 tehtävää: 2-numeroinen yhteenlasku ilman kymmenlukuun siirtymistä
Tulokset:
- 20 oppilasta: 5/5 oikein (hallitsee)
- 7 oppilasta: 3-4/5 oikein (kehittymässä)
- 3 oppilasta: 0-2/5 oikein (ei ymmärrä)

Toimenpide:
- Tiistai: Koko luokka etenee kymmenlukuun siirtymiseen
- Tiistai pienryhmä: 3 vaikeuksia kokevaa oppilasta saavat uudelleenopetusta
```

**Tiistain mini-tarkistus**:
```
5 tehtävää: 2-numeroinen yhteenlasku kymmenlukuun siirtymisen KANSSA
Tulokset:
- 15 oppilasta: 5/5 oikein (valmis etenemään)
- 10 oppilasta: 2-4/5 oikein (tarvitsee lisää harjoitusta)
- 5 oppilasta: 0-1/5 oikein (merkittäviä vaikeuksia)

Toimenpide:
- Keskiviikko: 15 oppilasta tekee itsenäistä harjoittelua (syventäminen)
- Keskiviikko: Opettaja työskentelee 15 oppilaan ryhmän kanssa
- Tukiopettaja ottaa 5 oppilasta intensiiviseen uudelleenopetukseen
```

**Hyöty**: Vaikeudet havaitaan ENNEN kuin niistä tulee aukkoja (reagoiva opetus)

## Virhekuvioiden analysointi

**Oikean/väärän lisäksi**: Analysoidaan MITEN oppilaat tekevät virheitä

### Matematiikan virhekuviot

**Kymmenlukuun siirtymisen virheet yhteenlaskussa**:

**Kuvio 1: Unohtaa siirtää kymmenlukuun**
```
  47
+ 28
----
  65  (väärin - oppilas laski 7+8=15, kirjoitti 5, unohti siirtää 1)

Diagnoosi: Ymmärtää yhteenlaskun, unohtaa kymmenlukuun siirtymisen vaiheen
Tuki: Visuaalinen kymmenlukuun siirtymisen harjoittelu (näytetään siirto selkeästi)
```

**Kuvio 2: Siirtää kymmenlukuun väärin**
```
  47
+ 28
----
  75  (väärin - oppilas siirsi 1 oikein mutta lisäsi sen väärään paikkaan)

Diagnoosi: Ymmärtää kymmenlukuun siirtymisen tarpeen, toteuttaa väärin
Tuki: Lukuarvon kertaus (mikä sarake on mikä?)
```

**Kuvio 3: Satunnaiset virheet (ei kuviota)**
```
Joskus oikein, joskus väärin (epäjohdonmukaista)

Diagnoosi: Huolimattomuusvirheet TAI työmuistin ylikuormitus
Tuki: Vähennä tehtävämäärää (vähemmän kerrallaan), tarkista työ -strategia
```

**Generaattorin etu**: Luo kohdennettua harjoittelua tietyille virhetyypeille
```
Luo moniste: 20 tehtävää, jotka nimenomaan vaativat kymmenlukuun siirtymistä
Fokus: Unohtaako oppilas AINA siirtää kymmenlukuun, vai vain joskus?
Data: Jos 0/20 oikein = taidon puute, jos 12/20 oikein = johdonmukaisuusongelma
```

## Kolmiportaisen tuen oppimisen seuranta

**Kolmiportaisen tuen viitekehys**: 3-tasoinen tukijärjestelmä

**Taso 1**: Yleisopetus (kaikki oppilaat)
**Taso 2**: Tehostettu tuki (vaikeuksia kokevat oppilaat, 20-30 % luokasta)
**Taso 3**: Erityinen tuki (merkittäviä vaikeuksia, 5-10 % luokasta)

---

### Tason 2 oppimisen seurantaprotokolla

**Tiheys**: Viikoittaiset monistetarkistukset (seurataan tuen tehokkuutta)

**Esimerkki: Oppilas, jolla vaikeuksia kertolaskun peruslaskuissa**

**Viikon 1 lähtötaso**:
```
Luo: 25 kertolaskua (0-10)
Oppilaan tulos: 8/25 oikein (32 %)
Päätös: Saa tason 2 tuen
```

**Viikot 2-8: Tuki + viikoittainen seuranta**
```
Viikko 2: Luo uusi moniste (25 laskua, sama taito)
         Tulos: 10/25 (40 %) - lievää edistymistä
         Jatketaan tukea

Viikko 3: Tulos: 12/25 (48 %) - jatkuvaa kasvua
         Jatketaan tukea

Viikko 4: Tulos: 11/25 (44 %) - lievä lasku
         Mukautetaan tukea (kokeillaan eri strategiaa)

Viikko 5: Tulos: 15/25 (60 %) - edistymistä muutoksen jälkeen
         Jatketaan tukea

Viikko 6: Tulos: 17/25 (68 %) - tasaista edistymistä
         Jatketaan tukea

Viikko 7: Tulos: 20/25 (80 %) - lähestytään hallintaa
         Jatketaan tukea (melkein perillä)

Viikko 8: Tulos: 22/25 (88 %) - hallinta saavutettu!
         Päätös: Poistutaan tasolta 2, palataan tasolle 1
```

**Generaattorin arvo**: Luo tuore arviointi joka viikko (ei ulkoa opettelua, todellinen taidon mittaus)

**Dokumentointi**: 8 monistetta kansiossa (todiste edistymisestä oppimissuunnitelma-/tukitiimille)

---

### Tason 3 oppimisen seurantaprotokolla

**Tiheys**: Kahdesti viikossa (intensiivinen seuranta)

**Lyhyemmät arvioinnit**: 10 tehtävää (vähemmän kuormittavaa)

**Esimerkki: Oppilas, jolla merkittäviä matematiikan puutteita**

**Viikko 1**:
```
Maanantai: 10 yhden numeron yhteenlaskua
           Tulos: 3/10 (30 %)

Torstai: 10 yhden numeron yhteenlaskua (eri tehtävät)
         Tulos: 4/10 (40 %) - edistymistä viikon sisällä!
```

**Viikko 2**:
```
Maanantai: Tulos: 5/10 (50 %)
Torstai: Tulos: 6/10 (60 %)
Trendi: Johdonmukaista kasvua
```

**Hyöty**: Kahdesti viikoittainen data näyttää tuen toimivuuden (tai toimimattomuuden tasannetilanteessa)

## Tietopohjaisen opetuksen kierto

**Periaate**: Käytä arviointidataa opetuksen jatkuvaan mukauttamiseen

### Viikkojakso

**Perjantai**: Analysoi viikon data
```
Tarkista kaikki päivittäiset 5-tehtävän tarkistukset:
- Ketkä oppilaat hallitsivat taidon? (valmiita etenemään)
- Ketkä oppilaat tarvitsevat lisää harjoitusta? (eivät aivan vielä)
- Ketkä oppilaat eivät ymmärtäneet? (tarvitsevat uudelleenopetusta)
```

**Sunnuntai**: Suunnittele seuraava viikko datan perusteella
```
Ryhmä 1 (hallitsee): Luo syventäviä monisteita (edistynyt soveltaminen)
Ryhmä 2 (kehittymässä): Luo lisää harjoitusmonisteita (sama taito, enemmän toistoja)
Ryhmä 3 (vaikeuksia): Luo mukautetut monisteet (yksinkertaistettu, tuettu)
```

**Maanantai-perjantai**: Tarjoa eriyttävää opetusta
```
Kaikki ryhmät: Työskentelevät sopivan tason materiaalien kanssa
Opettaja: Kiertää, antaa kohdennettua tukea tarpeen mukaan
```

**Seuraava perjantai**: Analysoi uudelleen, ryhmittele tarpeen mukaan

**Tulos**: Opetus vastaa jatkuvasti oppilaiden tarpeisiin


**Hyöty**: Oppilas näkee oman kasvunsa (motivaatio), opettaja dokumentoi edistymisen (vastuullisuus)

## Tehokkaat tarkistusstrategiat

**Ongelma**: 30 oppilaan päivittäinen tarkistaminen = aikataakka

**Ratkaisut**:

### Strategia 1: Oppilaan itsearvioin ti (vastausavaimella)

**Prosessi**:
```
1. Oppilaat tekevät 5-tehtävän tarkistuksen (5 min)
2. Opettaja näyttää vastausavaimen (1 min)
3. Oppilaat tarkistavat oman työnsä, merkitsevät X väärin (2 min)
4. Oppilaat kirjoittavat tuloksen ylös (30 sekuntia)
5. Palauttavat opettajalle

Opettaja: Silmäilee nopeasti tulokset (2 min), huomioi ketkä tarvitsevat tukea
Kokonaisaika: 10,5 minuuttia (vs 20+ minuuttia jos opettaja tarkistaa kaikki)
```

**Hyöty**: Oppilaan vastuu (näkee omat virheensä heti), ajantehokasta

---

### Strategia 2: Paritarkistus (parin kanssa)

**Prosessi**:
```
1. Oppilaat tekevät monisteen (5 min)
2. Vaihtavat parin kanssa
3. Opettaja näyttää vastausavaimen
4. Parit tarkistavat toistensa työt (2 min)
5. Palauttavat paperit, keskustelevat virheistä (1 min)
6. Palauttavat opettajalle

Kokonaisaika: 8 minuuttia, sisältää vertaiskeskustelun
```

---

### Strategia 3: Digitaalinen palautus (Google Classroom)

**Prosessi**:
```
1. Oppilaat tekevät monisteen (5 min)
2. Ottavat kuvan, lähettävät Google Classroomiin (2 min)
3. Opettaja tarkastaa digitaalisesti sopivana aikana (koulun jälkeen, suunnitteluaika)

Hyöty: Ei tuntiaikaa tarkistamiseen
```

## Tuen suunnittelu datan perusteella

**Data ohjaa tukipäätöksiä**

### Skenaario 1: Koko luokan vaikeudet

**Data**: 80 % luokasta sai alle 60 % perjantain tarkistuksessa

**Diagnoosi**: Opetus tehotonta (useimmat oppilaat eivät ymmärtäneet)

**Toimenpide**:
- **Maanantai**: Opeta KOKO käsite uudelleen (eri lähestymistapa)
- **Tiistai**: Harjoittelu (uusi moniste)
- **Keskiviikko**: Tarkista uudelleen (5-tehtävän arviointi)
- **Tulos**: Jos edelleen matalaa, kokeile TOISTA opetusmenetelmää (pyydä tukea)

---

### Skenaario 2: Pienryhmän vaikeudet

**Data**: 5 oppilasta johdonmukaisesti alle 60 %, muu luokka 85 %+

**Diagnoosi**: Pienryhmä tarvitsee kohdennettua tukea (ei koko luokan ongelma)

**Toimenpide**:
- **Pienryhmätuki**: 20 minuuttia päivittäin erityisopettajan tai tukiopettajan kanssa
- **Mukautetut monisteet**: Vähennetty tehtävämäärä (10 sijaan 20), kuvatuki
- **Päivittäinen oppimisen seuranta**: Seurataan näitä 5 oppilasta päivittäin (vs viikottain muille)

---

### Skenaario 3: Yksittäinen oppilas poikkeaa

**Data**: Yksi oppilas saa 0-20 % kun kaikki muut 70 %+

**Diagnoosi**: Merkittävä taidon puute TAI muu tekijä (näkö, tarkkaavuus, prosessointi)

**Toimenpide**:
- **Konsultaatio**: Oppilashuoltotiimi, mahdollinen erityisopetuksen arviointi
- **Välitön tuki**: Mukautetut materiaalit, 1-1-tuki
- **Diagnoottinen arviointi**: Tunnista MISSÄ ymmärrys katkesi (lukuarvo? lukumääräkäsitys?)
